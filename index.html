Project 1

For this project, I developed a personal portfolio website using the iPortfolio Bootstrap 5 Resume Template downloaded from htmltemplates.co . The template was selected because it offers a clean, professional, and structured layout that is well suited for a portfolio website. Since it is built using Bootstrap 5, it is fully responsive and automatically adapts to various screen sizes.

Rather than building a website completely from scratch, the objective of this project was to customize an existing template and adapt it to meet academic requirements. This required understanding how the template was structured, how Bootstrap classes controlled layout and responsiveness, and how different HTML and CSS components interacted with one another.

Homepage and Content Customization

The homepage content was fully customized to reflect my education and the course I am currently studying. The original placeholder text from the template was removed and replaced with relevant academic information. Additionally, text in the About and Resume sections was rewritten to represent my own background instead of the template example content.

What I Changed from the Original Template

The template underwent significant structural and visual modifications to transform it into a personalized academic portfolio website.

  • Removed the Portfolio and Testimonials sections as they were not required.
  • Modified the section originally called "downdeep" and restructured it into a new Projects section.
  • Created subsections within Projects including Project 1, Project 2, Project 3, and Final Project.
  • Created additional sections such as Process and Goals.
  • Replaced all template images with personal or royalty-free images.
  • Resized and optimized images to reduce loading time.
  • Customized homepage text to reflect my education and course.
  • Edited content in About and Resume sections.
  • Created a custom logo, app icon, and favicon using Canva.
  • Integrated Formspree instead of the original PHP contact form.
  • Adjusted CSS styling to preserve the visual theme after structural changes.

Design Decisions and Visual Theme

A key visual feature of the template is its alternating section design. Sections switch between white backgrounds and blue-tinted backgrounds, creating contrast and making each section stand out individually. I decided to preserve this design because it enhances readability and gives the page a clear visual structure.

However, after removing several original sections, the alternating pattern became inconsistent. To correct this, adjustments were made to the HTML structure and CSS styling to ensure that the alternating background pattern continued correctly throughout the page.

The fixed side navigation menu was also retained because it improves usability and allows users to quickly access each section of the website.

Image Replacement and Optimization

Challenges and Technical Problem Solving

One major issue occurred when creating the new Projects section. Initially, the navigation menu linked to a section called "Project" that did not exist. When clicked, the page redirected to the homepage because the ID could not be found.

This was resolved by creating a properly structured section with the correct ID and ensuring that the navigation menu href values matched exactly.

After adding the section, the template styling did not automatically apply. The design of the new section differed from the rest of the page. To fix this, modifications were made in both index.html and main.css to maintain visual consistency.

Another challenge involved the template’s scroll animation system. The template uses data-aos="fade-up" attributes to animate elements when they scroll into view. Newly created sections did not include these attributes, so the animation effect did not work.

After analyzing the code, I added the correct AOS attributes and delay values to each new subsection, restoring the smooth fade-up animation across the entire site.

Additionally, this was my first time using GitHub and GitHub Pages. Understanding repositories, commits, deployment, and file structure required experimentation and learning.

Uploading the assets folder caused difficulties due to size and structure limitations. The issue was resolved by first creating an empty assets folder in the repository and then manually uploading the necessary subfolders and files.

Since this was also my first experience coding in HTML and CSS, understanding how Bootstrap classes affected layout and responsiveness required patience and practice.

Throughout the project, ChatGPT was used as a learning tool to explain unfamiliar coding concepts and assist in debugging issues. Rather than copying code without understanding it, I focused on learning the logic behind the solutions.

Contact Form and Hosting Limitations

The original template included a PHP-based contact form. However, GitHub Pages does not support server-side PHP execution. This meant the contact form could not send emails directly.

To overcome this limitation, the form was integrated with Formspree. Formspree processes form submissions externally and allowed messages to be received through the Formspree dashboard, including name, email, subject, and message.

Project 2

Wall-Mounted Shelf

This project was part of my work in computer controlled fabrication, where the goal was to design and manufacture an object using digital tools and a laser cutter. The process involved designing parts in CAD software, testing material tolerances, and preparing files for laser cutting.

My original idea was to design a small wall-mounted shelf inspired by designs I found on Pinterest. I began modeling the shelf in Fusion 360 and experimented with parametric design and finger joints for assembly. During the design process I encountered several challenges related to parameters, part dependencies, and laser cutting settings such as kerf compensation.

After multiple design iterations, I decided to change direction and instead create a decorative lamp made from black plexiglass. The lamp uses lasercut patterns that allow light to pass through the material and create shadow silhouettes. The project required testing kerf values, designing finger joints, preparing vector graphics, and assembling the final object with electrical components.

This project demonstrates the full workflow from idea and digital design to testing, fabrication, and final assembly using laser cutting and digital fabrication tools.

Idea and Brainstorming

At the beginning of the project I decided to design a small wall-mounted shelf. I wanted to create something simple but functional that could be assembled using press fit joints instead of screws or glue. The idea was to explore how laser cutting could be used to create structural parts that fit together precisely. The inspiration for the shelf came from Pinterest The sweetest shelves , where I found several minimal wall shelf designs. These references helped me think about the overall proportions and structure of the shelf, but I planned to redesign the object so that it could be fabricated using finger joints and parametric modeling.

For the design work I used Autodesk Fusion 360. I had already used Fusion in another course related to computercontrolled fabrication, so I was somewhat familiar with the workflow. Fusion is developed by Autodesk, which is the same company that makes AutoCAD, a program I had previously used for technical drawing. Because of that, the design environment felt familiar and it was relatively easy to start modeling.

The goal was to design the shelf parametrically, meaning that the most important dimensions—such as material thickness, shelf width, and height could easily be adjusted without breaking the model.

Design

My first attempt at designing the shelf did not go very well. I tried to implement parameters early in the modeling process, but I did not structure them correctly. Because of this, the relationships between different parts of the model became confusing and the design quickly turned into a messy collection of sketches and features.

When I tried to change certain dimensions, other parts of the model did not update correctly. This defeated the purpose of using parametric design in the first place. Eventually the model became difficult to modify and I decided it would be easier to start over rather than trying to repair the original design. I then created a second version of the shelf, where I took more time to carefully define the dimensions and parameters. I tried to think more systematically about how the parts were connected and how the dimensions should depend on each other. This second attempt worked significantly better. The sketches were more organized and the parameters were clearly defined, which made it easier to adjust the overall size of the design.

Parametric Challenges

Even though the second version of the shelf worked better, I still ran into some issues with the parametric behavior of the model. For example, when I changed the length of one part of the shelf, the other connected parts did not follow automatically and were instead left floating in space. Ideally, changing one parameter should propagate through the entire design, but in my case some components remained unchanged and the geometry no longer matched.

After discussing the problem, I learned that it can be helpful to create a master sketch that controls the most important dimensions of the design. This master sketch acts as the main reference for the rest of the model. Individual parts can then reference geometry from that sketch, ensuring that changes propagate consistently throughout the design.

After spending quite a bit of time struggling with this and implementing a master sketch, I also realized something important about the workflow. While it is possible to fully align all the parts inside the 3D model, it is not always necessary. Since the parts will ultimately be laser cut and laid flat on the cutting bed, they do not actually need to be positioned together in a fully assembled way inside the design file.

Another challenge involved the material thickness and finger joints. Since the parts were designed to fit together using finger joints, the thickness of the joints needed to match the material thickness exactly. In some parts of the model I had forgotten to define the thickness parameter for the finger joint geometry. As a result, the joints did not match the intended dimensions. To correct this issue, I had to recreate some of the parts and properly define the thickness parameter so that all joints would remain consistent if the material thickness changed.

Laser Cutting Setup Problem

Once the design was complete, I moved on to preparing the file for laser cutting inside Autodesk Fusion 360. I created a laser cutting setup and attempted to generate the cutting paths for the model.

At first, something seemed wrong. The software correctly detected the holes for the finger joints along the sides of the shelf, but it refused to recognize the finger joint holes located in the center of the top plate. Because of this, those holes were not included in the cutting operation.

After investigating the issue, I discovered that the problem was related to the kerf setting. The kerf value had been set to 4 mm, which is extremely large for a laser cutting offset. Because of this incorrect value, the software interpreted some of the internal geometry incorrectly and refused to generate the correct cutting paths.

At first I thought this value came from a tutorial video I had watched. However, after checking the video again, I realized that the 4 mm measurement actually referred to the nozzle size, not the kerf value. I had mistakenly assumed it was the kerf setting and entered it into the software. Once I corrected the kerf value to a realistic number, the cutting paths were generated correctly and the problem was resolved.

Decorative Lamp

New Idea

Instead of continuing with the shelf, I decided to design a decorative lamp that could be laser cut from acrylic. There were several reasons for this change:

  • The 5 mm plexiglass sheets were out of stock
  • The lab still had 3 mm black plexiglass available
  • I found an interesting lamp concept on Pinterest
The sweetest shelves

After deciding to move away from the shelf idea, I began working on a decorative lamp design. The goal was to create something that could take advantage of the visual qualities of laser-cut acrylic. I wanted the object to not only function as a lamp but also create interesting light and shadow patterns.

The lamp was designed as a simple finger joint box structure made from five parts: four side panels and one bottom panel, that is fasten with stab through finger joints through the side panels. I intentionally left the top open so the light could escape upward while the cut patterns on the sides would project shadows into the surrounding space. The parts were connected using finger joints, which allowed the lamp to be assembled without screws or glue. The design was made to be simple and structurally stable while still allowing the decorative patterns to be the main visual element.

To create the decorative patterns, I used Inkscape. I found images online that I wanted to turn into silhouettes and imported them into Inkscape. Using the Trace Bitmap tool, I converted the images into vector graphics. Vector graphics are necessary for laser cutting because they define precise cutting paths rather than pixel-based images. After tracing the images, I exported the designs as SVG files. These SVG files were then imported into sketches in Autodesk Fusion 360. Inside Fusion, I positioned the designs on the walls of the box and used them to create cut-outs in the panels. I chose to remove the background shapes so that the light source itself becomes the background, while the figures remain as solid silhouettes that block the light.

Lamp design

To create the patterns on the sides of the lamp, I searched for visual inspiration on Pinterest. I wanted silhouettes that would look interesting when illuminated from the inside. Once I found suitable images, I used Inkscape to convert them into vector graphics. Raster images cannot be used directly for laser cutting, so they first needed to be transformed into vector paths.

In Inkscape I used the Trace Bitmap tool, which analyzes the image and converts it into vector outlines. After adjusting the traced shapes and cleaning up the paths, I exported the designs as SVG files. The SVG files were then imported into Autodesk Fusion 360, where they were placed into sketches on the side panels of the lamp. From there I used sketch profiles to create cut-outs in the panels.

Instead of cutting out the silhouettes themselves, I chose to remove the background and leave the figures solid. This way, when the lamp is turned on, the light shines through the background areas while the figures appear as dark silhouettes.

The lamp was fabricated using 3 mm black acrylic (plexiglass). This material works well for laser cutting because it produces clean edges and allows light to pass through the cut areas. Originally I had planned to work with 5 mm plexiglass, but that material was no longer available in the lab. Because 3 mm black acrylic was still in stock, I adapted the design to work with that thickness instead. Using darker acrylic also helped emphasize the light effect, since the contrast between the illuminated cut-outs and the dark material creates stronger shadow patterns.

Kerf and Joint Testing

Since the material had changed from the earlier design attempts, I needed to perform a new kerf test before cutting the final parts. Kerf refers to the small amount of material removed by the laser beam during cutting. Even though the laser beam is very thin, it still removes a measurable amount of material. If this offset is not taken into account, press-fit joints can end up either too loose or too tight. To find the correct settings, I created several test pieces to experiment with different kerf values and joint sizes. These tests included both finger joints and stab-in joints, which are small tabs that slide into corresponding slots. In total I produced seven different test pieces, adjusting the cutting parameters each time. After evaluating how tightly the pieces fit together, I found that the best settings were:

  • Kerf: 0.17 mm
  • Speed: 22%

With these settings the finger joints were tight enough to hold the structure together, but still possible to assemble without damaging the acrylic. The stab-in joints also fit securely into their slots.

Cutting and Assembly

Once the design and settings were finalized, I proceeded to laser cut the final parts. The lamp consisted of five pieces: four side panels containing the decorative patterns and one bottom panel. The cutting process went smoothly, and the laser successfully cut through the acrylic without any issues. The detailed patterns were reproduced very well, and the edges of the acrylic came out clean. Some other students experienced problems where the laser did not fully cut through their material, but in my case the chosen settings worked reliably.

During assembly I discovered that one of the side panels was missing the slots for the bottom panel. This was a design oversight, so that particular panel had to be laser cut again. When quickly adding the missing slots, I accidentally placed them slightly too high in the design. Because of this, the finger joints on the sides did not align perfectly. Instead of cutting yet another panel, I decided to correct the problem manually. I used sandpaper to slightly adjust the finger joints until the parts fit together properly. Since there was a limited amount of 3 mm black acrylic available, this was the safest solution.

After all the panels fit together correctly, I assembled the lamp structure and installed the electrical components. I purchased the following items from a store:

  • A power cord with an on/off switch
  • A light socket
  • A cable for connecting the components

Once these parts were installed inside the acrylic box, the lamp was complete. When turned on, the light shines through the cut-out patterns and creates shadow silhouettes on the surrounding surfaces. The final result is a decorative laser-cut lamp that combines digital design, material testing, and fabrication.

Project 3

Project Idea

I decided to design something useful for my car. I ended up creating a trash can, but not just a regular one. This trash can is designed to perfectly fit a specific compartment in my car. The compartment is located where an armrest is often placed in many cars. In my car it doesn't really serve any purpose except collecting trash, so I thought it would be a perfect place for a small custom trash can.

Measuring the Compartment

The first step was to recreate the bottom and top shape of the compartment. I first tried removing the mat inside the compartment, but that was not possible. After that I attempted to measure the compartment directly, but since the shape is irregular it was very difficult to measure accurately.

Instead, I decided to take photos of the compartment with a measuring tape visible in the images. This allowed me to estimate the correct scale and trace the shape digitally. I did this for both the top and the bottom of the compartment.

First Prototypes

I printed my first prototypes using a Creality K2 3D printer. The pieces almost fit, but both of them were slightly too long. I believe this happened because the photos were not taken perfectly from above, which caused some perspective distortion when scaling the shapes.

To fix this, I made a cardboard version of the shape by tracing the printed prototype onto corrugated cardboard. I cut it out and manually adjusted the shape until it fit perfectly inside the compartment. Then I scanned the cardboard template and created a second prototype which I printed on a Prusa printer in VR-III. This prototype fit perfectly.

Designing the Trash Can

The compartment is not equally deep everywhere, so I needed to calculate the correct angle for the top surface. I measured the depth at several different locations and recreated these measurements in Fusion. Using these measurements I tilted the top plane to match the slope of the compartment.

Once the correct angle was established, I created the walls of the trash can and added a bottom surface. The walls were extended 1 cm above the compartment to create a visible rim. I also designed a lid with a circular swing door.

The swing lid was inspired by this design: Trash Can With Swing Lid .

Adding Text with Inkscape

To add a personal touch, I added text to the bottom of the trash can that says "Rusl Rakelar".

To do this I used Inkscape. I created the text and converted it using Text → Text and Font → Text to Path → Object to Path. The file was then exported as an SVG file and imported into a sketch in Fusion where I extruded the text into the model.

Preparing for 3D Printing

After confirming that all parts worked correctly, I aligned the entire model with the XY plane and exported it as a 3MF file. The file was then imported into the Creality K2 slicer.

Because some edges of the lid extend outward, I needed to generate support structures to ensure the print would succeed.

What I Learned

  • Measuring irregular shapes can be difficult and often requires multiple techniques.
  • Photos can help estimate scale, but camera angle can introduce distortion.
  • Test prints are extremely useful for refining dimensions.
  • Cardboard prototypes can save time and plastic during early iterations.
  • SVG files from Inkscape can be imported into Fusion to add custom text or graphics.
